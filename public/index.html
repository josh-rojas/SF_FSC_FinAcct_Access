Employee Financial Account Access Management

A secure solution for controlling employee access to financial accounts within Salesforce Financial Services Cloud.

Key Features

Three-Tier Permission Structure

Granular control with base, mid-level, and admin access permissions for different user roles.

Employee Account Flagging

Automatic identification and special handling of financial accounts owned by employees.

Access Management Dashboard

Lightning Web Component interface for managing user permissions and access overrides.

Solution Architecture

Data Model

  • Custom field on User object: Is_Employee_Account__c (Boolean)
  • Custom object: FinancialAccountAccess__c
  • Custom metadata: AccessLevelSetting

Permission Structure

  • Base Level: No access to employee financial accounts
  • Mid Level: Selective access based on role hierarchy
  • Admin Level: Full access for compliance oversight

Deployment Guide

Deployment Instructions

  1. Authenticate with Salesforce using: sf org login
  2. Create a scratch org: sf org create scratch -f config/project-scratch-def.json -a FSCloud
  3. Deploy the components: sf project deploy start
  4. Open the app: sf org open -p /lightning/app/Financial_Services_Cloud_App
  5. Assign permission sets to users based on their access level requirements
